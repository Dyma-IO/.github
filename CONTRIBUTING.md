# Contributing to Dyma

Thank you for contributing to [Dyma](https://dyma.io).

## Where to start

| Goal | Channel |
|:-----|:--------|
| Bug reports and feature requests for a specific repo | Open an **issue** in that repository |
| Questions, ideas, and community chat | [Organization Discussions](https://github.com/orgs/Dyma-IO/discussions) |
| Code changes | Pull request in the relevant repository |
| Security vulnerabilities | Email [dymaweb3@gmail.com](mailto:dymaweb3@gmail.com) privately |

## Code workflow

1. Open an issue or comment on an existing one before large changes.
2. Fork the repository and create a branch from `main`.
3. Keep commits focused; write clear commit messages.
4. Open a pull request against `main` with a concise description and test plan.
5. Ensure CI passes before requesting review.

## Standards

- Match existing code style and conventions.
- Add or update tests for behavior changes.
- Do not commit secrets, `.env` files, or credentials.
- Reference issue numbers in PR descriptions when applicable.

## Discussions etiquette

- Search existing threads before posting.
- Use the category that best matches your topic.
- For Q&A, mark helpful answers when your question is resolved.
- Keep announcements and roadmap threads focused; use General or Ideas for open conversation.

## Security

Report vulnerabilities privately to **dymaweb3@gmail.com**. Do not open public issues or discussions for security findings.
